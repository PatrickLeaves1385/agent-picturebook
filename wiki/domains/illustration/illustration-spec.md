# 绘本插图通用规范（illustration-spec.md）

## 定位与优先级

本规范是**跨项目通用的兜底基线**，不是强制标准。

| 场景 | 约束来源 | 说明 |
|---|---|---|
| 项目有具体插图要求（`wiki/projects/{project_id}/` 中的 content-spec / characters / 视觉风格页） | **项目要求优先** | 冲突时一律以项目要求为准 |
| 项目无具体插图要求，或要求未覆盖某项 | **回退到本规范对应章节** | 本规范作为通用基线补位 |
| `raw/` 下用户放置的风格参考文件 | 作为本项目实证参考叠加 | 与本规范冲突时，项目实证参考优先 |

> 约束解析顺序：项目要求 > raw/ 风格参考文件 > 本规范（通用兜底）

illustration-agent、illustration-generate Skill、illustration 类质检项在执行时，按此顺序解析约束。任何环节检测到冲突，应标注 `[!] 项目要求与通用规范冲突：{说明}`，并按项目要求执行。

---

## 1. 画面基础参数

| 参数 | 通用基线 | 备注 |
|---|---|---|
| 画幅比例 | 竖版 768×1024（绘本主流）或 方版 1024×1024 | 项目可指定横版 1024×768 |
| 分辨率 | ≥1024px 长边 | 满足印刷与屏幕双用途 |
| 色彩模式 | RGB 生成，印刷用途时转 CMYK | 生成阶段统一 RGB |
| 输出格式 | PNG（无损，支持版本化命名） | — |

项目有指定画幅时，以项目要求为准。

---

## 2. 风格基线（兜底参数）

当 `raw/` 无风格参考文件、wiki 中无项目级视觉风格设定时，使用以下通用儿童绘本风格参数作为兜底：

| 维度 | 兜底参数 |
|---|---|
| 色调 | 明亮温暖，柔和光影 |
| 线条 | 圆润、手绘感 |
| 氛围 | 轻快、安全、友善 |
| 质感 | 柔和过渡，避免锐利边缘与强对比阴影 |
| 造型 | 角色比例偏 Q 萌（大头小身），符合低龄审美 |

> 此为兜底，非默认推荐。优先使用项目级风格设定或 raw/ 风格参考文件。使用兜底参数时，illustration-agent 须标注 `[i] 使用通用风格基线，建议补充项目风格参考以提高一致性`。

## 3. 构图规范

### 3.1 单页构图原则

| 原则 | 说明 |
|---|---|
| 主体聚焦 | 主体居中或遵循三分法，避免边缘切角 |
| 留白充足 | 主体周围留白，画面不拥挤，便于儿童识别 |
| 视觉引导 | 用色彩对比/线条方向引导视线至主体 |
| 视角统一 | 同一场景连续页面保持视角一致，避免突兀切换 |

### 3.2 跨页连续性

- 同一场景的连续页面，色调与构图须自然过渡
- 角色在同场景多页出现时，体型比例与位置关系保持一致

## 4. 插图描述格式（脚本侧规范）

脚本中每页的「插图描述」字段应遵循以下格式，供 illustration-agent 解析生成。此格式同时是质检项 `illustration/composition` 的检测基准。

### 4.1 必备 5 项

| # | 规范 | 要求 | 质检对应 |
|---|---|---|---|
| 1 | 只描述静态画面 | 描述一个定格瞬间，不写动作过程（禁止 "first...then..." 等过程描写） | composition #1 |
| 2 | 标注景别与视角 | 含全景/中景/近景/特写 之一 + 平视/仰视/俯视 之一 | composition #2 |
| 3 | 区分前中后景 | 明确前景/中景/后景（或 foreground/midground/background） | composition #3 |
| 4 | 写明每个角色姿势与朝向 | 每个出场角色标注姿势（站/坐/蹲/侧身…）+ 朝向（面朝观众/侧身朝左…） | composition #4 / character-pose |
| 5 | 纯静物页注明 | 全页无角色时，注明「本页为纯静物特写」 | composition #5 |

### 4.2 推荐写法

```
**插图描述**：
- 景别/视角：{全景/平视}
- 前景：{角色名}，{姿势}，{朝向}，{表情}
- 中景：{场景元素}
- 后景：{环境/氛围}
- 情绪基调：{明亮温暖/宁静/欢快}
```

### 4.3 豁免

- 纯画面页（无文本/无角色）允许只满足 #5
- 收尾页允许简化描写
- 远景非对话路人角色可豁免姿势朝向（见 character-pose 质检项）

> 项目级 content-spec 如有自定义插图描述格式，以项目为准；本格式仅在项目未定义时作为兜底。

## 5. 角色表现规范

| 维度 | 通用基线 | 项目可覆盖 |
|---|---|---|
| 造型一致性 | 核心角色在所有页面体型比例、特征统一 | 项目 characters.md 定义优先 |
| 表情约束 | 明亮正向，禁止恐惧/哭泣/愤怒/无助 | 项目可放宽或收窄 |
| 姿势朝向连续性 | 同角色连续多页至少一页完整描写姿势+朝向 | — |
| 道具一致性 | 角色专属道具（IP 钩子）跨页视觉统一 | 项目 IP 钩子定义优先 |

---

## 6. 禁止项（恒定底线）

以下为**恒定禁止项**，任何项目不得放宽（项目可在此基础上追加更多禁止项，但不可移除以下底线）：

- 禁止出现任何文字、字母、数字（绘本图为纯视觉，文字属脚本侧）
- 禁止恐怖、阴暗、吓人元素
- 禁止锐利阴影、强对比暗色调
- 禁止角色表情为恐惧/哭泣/愤怒/无助

> 项目可在 wiki/projects 中追加项目专属禁止项（如特定符号、特定场景），illustration-agent 读取后并入 prompt 禁止层。

## 7. 版本化与命名

图片文件命名遵循全项目统一规则（权威定义见 CLAUDE.md 硬性规则 #9）：

```
ep{集号}_p{页号}_v{版本号}.png
```

- 集号 `ep{X}`：故事/单集编号
- 页号 `p{Y}`：脚本内页码
- 版本号 `v{Z}`：该页图片版本，从 v1 起，修改后递增，**永不覆盖旧版本**

元数据记录在同目录 `_metadata.json`（格式见 illustration-generate Skill Step 5）。

## 维护说明

本规范支持用户持续更新。维护规则：

1. **修改方式**：直接编辑本文件对应章节。修改后递增 frontmatter 的 `version`（v1 → v2 → …），更新 `last_updated`。
2. **变更记录**：在 `wiki/log.md` 追加一行，说明改了哪一章、为什么改。
3. **向后兼容**：修改时注意不要破坏既有项目的已生成图片（版本化文件永久保留，规范变更不影响历史产物）。
4. **章节独立**：各章节设计为可独立增删，新增章节追加到末尾并递增章节号，不重排已有编号。
5. **禁止项只增不减**：第 6 节恒定禁止项只能追加，不可移除已有底线。
