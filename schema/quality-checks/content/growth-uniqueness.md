# 主题唯一性检测

对比新故事的核心主题与已有故事。

## 量化锚点（v2 新增）

每个已有故事按以下计算与新故事的主题相似度：

### 维度 1：关键词重合度（自动统计）

| 信号 | 来源 |
|---|---|
| 提取新故事核心主题词（标题 + 梗概 + 成长点） | 脚本 / 创意稿 |
| 提取已有故事核心主题词 | wiki-context 中 episodes/stories 页面 |
| 计算关键词集合的 Jaccard 相似度 | 自动统计 |

Jaccard = 交集 / 并集

### 维度 2：成长点重合度（LLM 审查）

- 提取新故事成长点（1-2 句）
- 与已有故事成长点对比
- LLM 打 0-1 分（1=完全相同，0=完全不同）

### 综合相似度

```
综合相似度 = 0.5 × 关键词Jaccard + 0.5 × 成长点LLM分
```

| 综合相似度 | 判定 |
|---|---|
| <= 60% | ✅ PASS |
| 60-80% | ⚠️ WARNING，标注相似故事 |
| > 80% | ❌ FAIL（高度重复，建议改角度） |
| = 100% | ❌ FAIL（完全相同） |

## 豁免

- 系列作品允许同主题不同角度（角度差异 >= 30% 时判定为 PASS）
- 角度差异由 LLM 审查（综合相似度 60-80% 时启用角度审查）

## 检测方法

1. 从 wiki-context 提取已有故事列表及其主题/成长点
2. 维度1 自动统计（Jaccard）
3. 维度2 LLM 审查
4. 按量化规则综合判定

## 判定（原）
- 主题与已有不重复 → PASS
- 高度相似 → WARNING，标注相似故事，建议调整角度
- 完全相同 → FAIL

## 注意
- 已有故事列表完全从 wiki-context 提取，不硬编码
- 系列作品允许同主题不同角度，判定时需考虑角度差异
- 无已有故事 → SKIP
