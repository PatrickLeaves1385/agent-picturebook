# 角色一致性检测

## 量化锚点（v2 新增）

每个出场角色按以下 3 维度量化打分：

### 维度 1：语言风格匹配度

| 规则 | 来源 |
|---|---|
| 提取 wiki 中角色"语言风格"或"性格"页的关键词 | characters.md / content-spec.md |
| 统计脚本中该角色台词是否包含风格关键词 | 全文 Grep 角色名 + 附近 50 字符 |
| 比例 = 命中次数 / 角色台词总数 | 自动统计 |

- 比例 >= 80% → PASS
- 比例 50-80% → WARNING
- 比例 < 50% → FAIL

### 维度 2：性格行为一致

| 行为模式 | 匹配信号 |
|---|---|
| 勇敢 | 主动尝试类动作/台词 |
| 害羞 | 犹豫语气/低头/慢动作 |
| 善良 | 关心他人/主动分享 |

- 提取 wiki 中角色性格特征关键词
- LLM 审查脚本中该角色行为是否符合（每行为 0/1 打分）
- 符合率 >= 80% → PASS
- 符合率 50-80% → WARNING
- 符合率 < 50% → FAIL（严重偏差）

### 维度 3：身份关系一致

- 角色身份（年龄/角色/关系）跨脚本多页一致
- 角色间关系（如 A 是 B 的朋友）跨页一致
- 不一致 → WARNING（标注具体页码和冲突）

## 判定综合

| 结果 | 条件 |
|---|---|
| ✅ PASS | 3 维度全 PASS |
| ⚠️ WARNING | 任意维度 WARNING，无 FAIL |
| ❌ FAIL | 维度2 FAIL（严重偏差）或多个维度 WARNING |

## 检测方法

1. 从 wiki-context 提取角色设定（名称/性格/语言风格/行为模式）
2. 维度1/3 用 Grep 自动统计 + 邻近关键词分析
3. 维度2 用 LLM 审查（语义判断最关键）

## 原方法保留

- 从 wiki-context 中提取角色设定（名称/性格/身份/语言风格/行为模式）
- 检查脚本中每个出场角色的行为和台词是否符合设定
- 如 wiki 中无角色设定，此检测自动跳过

## 判定（原）
- 所有角色符合设定 → PASS
- 偏差（语气不完全匹配等）→ WARNING
- 严重偏差（性格反转/设定冲突）→ FAIL

## 注意
- 不做跨项目的角色假设
- 角色设定完全依赖 wiki-context 中的内容
- 维度1/3 的关键词词表由项目扩展
- 无角色设定 → SKIP
